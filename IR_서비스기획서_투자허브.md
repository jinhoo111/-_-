## 1. Executive Summary (한 장 요약)

**투자 허브는 흩어진 투자 정보 노동을 하나의 대시보드로 통합하는 웹 서비스입니다.**

개인·기업 투자자는 시세·공시·뉴스·규제·기록을 각기 다른 5~6개 채널(증권 앱, DART, 뉴스 포털, 엑셀, 메신저)을 오가며 수집합니다. 투자 허브는 이 파편화된 흐름을 **① 다계좌 포트폴리오 ② 기업/경쟁사 공시 모니터링 ③ 실시간 지수·환율 ④ 뉴스·월가 레이팅 ⑤ 규제 모니터링 ⑥ 투자 일지(가계부·건강까지)** 의 단일 화면으로 묶었습니다.

- **차별점 한 줄:** "종목 관리 앱"이 아니라 **"투자 의사결정에 필요한 외부 정보를 먼저 찾아와 정리해 주는 브리핑 레이어"** 입니다.
- **수익 구조:** 신호(공시·시세·알림)는 무료, **AI 해석·자동화·리포트(경쟁사 브리핑)는 Pro 유료** — 특히 상장사 IR/전략기획팀의 "경쟁사 모니터링 노동 제거"를 지불 이유로 설정.
- **현 상태:** 단일 페이지 앱(SPA)으로 **실제 배포·운영 중**. Supabase 기반 서버 인프라, 보안 강화(JWT 검증·CORS 화이트리스트·XSS 방어), 디자인 시스템 토큰화 완료.
- **투자 요청:** 아래 §12 로드맵(공용 AI 키·리포트 내보내기·이메일 다이제스트·결제 게이트) 실행 및 B2B 세일즈 검증을 위한 시드 자금.

## 2. 문제 정의 (Problem)

### 2-1. 개인 투자자
- 시세는 증권 앱, 공시는 DART, 뉴스는 포털, 기록은 카카오톡/엑셀에 흩어져 있음.
- **정보 수집에 드는 시간 > 실제 판단에 쓰는 시간.** 매매 근거가 기록되지 않아 같은 실수를 반복.
- 미국 주식 병행 시 환율·프리마켓·월가 레이팅까지 챙겨야 해 정보 부하가 급증.

### 2-2. 기업 투자자 / 상장사 (B2B 핵심)
- IR·전략기획팀은 **경쟁사 공시·지분변동·규제 동향**을 사람이 매일 손으로 수집해 주간 보고서를 만듦 → 반복적·소모적 노동.
- DART/KRX 데이터는 있지만 **"의미 있는 변화만 골라 브리핑"** 해 주는 도구가 없음.

### 2-3. JTBD (Job-to-be-Done)
> **개인:** "장 마감 후 내 포트폴리오를 점검할 때, 흩어진 정보를 한 번에 보고 싶어서, 5분 안에 오늘 무슨 일이 있었는지 파악하고 다음 액션을 기록하고 싶다."
>
> **기업:** "월요일 아침 경영진 보고를 준비할 때, 경쟁사에 생긴 중요한 변화만 골라, 손으로 뒤지지 않고 요약된 브리핑으로 받고 싶다."

**현재 대안(hire)과 한계(fire 지점):** 증권사 HTS(자기 종목만·외부 정보 약함), DART 원문(가공 안 됨), 뉴스 구독(개인화 없음), 엑셀 수기(자동화 없음). → **"통합 + 자동 정리"의 공백**이 기회.

## 3. 솔루션 (Solution)

**"투자 판단에 필요한 외부 정보를 한 화면에 모으고, 반복 노동을 자동화한다."**

7개 기능 축이 하나의 SPA로 통합되어 있으며, 개인/기업 계정 유형에 따라 온보딩·기능 노출이 분기됩니다.

| # | 기능 축 | 핵심 가치 | 데이터 소스 |
|---|---|---|---|
| 1 | **기업 모니터링 (경쟁사 레이더)** | 관심/경쟁 기업 공시·지분변동·보유주식 추적, 전략적 신호 분류 | DART (corp_code 변환) |
| 2 | **투자 종목 관리** | 다계좌·원화/달러 통합 포트폴리오, 실시간 손익, 종목 소식 피드 | Finnhub, Naver, Yahoo |
| 3 | **지수·환율** | 국내·미국 지수, VIX, 선물, 환율, 가상화폐 한눈에 | Yahoo Finance, CoinGecko |
| 4 | **뉴스** | 시장/종목별 뉴스 + 월가 레이팅 + 실시간 시세, 원클릭 일지 전환 | Finnhub, Naver |
| 5 | **리서치·규제** | 美(SEC·FTC·Fed 등)·韓(금융위·DART 등) 규제 동향 모니터링 | 규제기관 피드 |
| 6 | **투자 일지** | 캘린더·아카이브·투자 철학·리포트(가계부·건강 트래킹) | 자체 DB |
| 7 | **보안 대시보드** | 이상행위 이벤트 모니터링 (관리자 전용) | 자체 로깅 |

**킬러 기능:** 기업 모니터링의 **"이번 주 경쟁사 브리핑"** — 신호를 먼저 찾아와 AI가 3줄로 요약(Pro).

## 4. 제품 상세 & 완성도

### 4-1. 이미 구현·배포된 것 (검증 완료)
- **다계좌 포트폴리오**: 21종목 이상 실운영, 총평가금액 원화/달러 토글, 종목별 손익·비중·상태(보유/매수/관심/숨김) 관리.
- **DART 공시 연동**: 3대 제약(TLS/CORS 차단·corp_code 변환)을 다중 프록시 경쟁(Promise.any) + corp_map으로 우회. 보유주식 현황·연간 보유 변동 이력 추정.
- **국내 시세 정확도**: KOSPI/KOSDAQ 시장 구분 오류(.KS 고정 버그) 해소 — Naver 우선 + Yahoo 폴백.
- **투자 일지**: 캘린더에서 날짜 클릭 → 즉시 기록, 어닝콜(美 Finnhub + 韓 DART 잠정실적)·FOMC·CPI 자동 표기, 가계부(목표 대비 실제 SVG 차트)·몸무게 트래킹까지 라이프 로그 확장.
- **투자 철학**: 일지를 "지켜야 할 것 / 절대 금지"로 승격해 개인 원칙 컬렉션화.

### 4-2. 안정성·보안 (엔터프라이즈 신뢰 기반)
- **API 키 서버 이관**: 브라우저 평문 저장 → Supabase Edge Function 경유, AES-256-GCM 암호화, 계정별 스코프.
- **JWT 서명 검증(fail-closed)**: 위조 신원(사칭) 차단. CORS 화이트리스트로 도메인 제한.
- **XSS 전수 방어**: 사용자 입력 렌더 지점 전체 이스케이프 처리, 안전 URL 검증.
- **디자인 시스템 토큰화**: 90개 디자인 토큰(색상·타이포·간격), WCAG 2.1 접근성(터치타겟 44px·포커스링·aria-label), 다크모드 대응.

### 4-3. 진행 중 (로드맵 §12에서 상술)
- 보안 이벤트 대시보드: 스키마·로깅·화면 완료, **프로덕션 DB 마이그레이션 적용 대기**.
- 경쟁사 모니터링 Pro 자동화(공용 AI 키·PDF/노션 내보내기·이메일 다이제스트·결제 게이트): 승인·설계 완료, 개발 대기.

## 5. 차별점 & 진입장벽 (Moat)

1. **정보 통합 폭:** 시세·공시·뉴스·규제·기록을 한 앱에 묶은 서비스는 드묾. 대부분 "종목 관리" 또는 "뉴스"에 특화.
2. **"먼저 찾아오는 브리핑" 관점:** 사용자가 검색하는 게 아니라, 서비스가 의미 있는 변화를 골라 요약. B2B 지불 의사와 직결.
3. **한·미 동시 커버리지:** DART(韓) + Finnhub/SEC(美)를 하나로. 국내·해외 병행 투자자 대상 실질적 우위.
4. **축적되는 데이터 자산:** 투자 일지·철학·경쟁사 관심군은 사용할수록 쌓여 **전환비용(lock-in)** 을 만듦.
5. **기술적 해자:** DART/시세 API의 CORS·시장구분·인증 제약을 이미 실전에서 해결(다중 프록시 경쟁, Naver 폴백, JWT 검증) — 후발주자가 반복 학습해야 하는 노하우.

**신호는 무료, 해석·자동화는 유료** — 경계가 자연스럽고 지불 이유가 뾰족함.

| 구분 | 무료 (Free) | Pro (유료) |
|---|---|---|
| 공시·시세·뉴스·규제 신호 | ✅ | ✅ |
| 레이더 피드·카드 배지·브라우저 알림 | ✅ | ✅ |
| **AI 경쟁사 브리핑(3줄 요약)** | ✖ (티저·잠금) | ✅ |
| **주간 AI 종합 리포트** | ✖ | ✅ |
| **PDF/노션 내보내기·이메일 다이제스트** | ✖ | ✅ (로드맵) |

- **현재 구현:** `_isPro = 승인 기업계정 || 관리자` 프록시로 게이팅(실 결제 전 단계). 개인계정은 잠금 티저 + 업그레이드 CTA 노출.

## 8. 시장 규모 (TAM / SAM / SOM)

> ⚠️ 아래는 **Bottom-up 가정 기반 추정치**입니다. 각 계수는 실사 시 원출처(통계청·KRX·업계 IR) 재검증 대상이며, 현 시점 확정 수치가 아닙니다.

### 산식
```
TAM = 잠재 사용자 수 × 연간 평균 지불 금액(ARPU)
SAM = TAM × 도달 가능 비율(10~30%)
SOM = SAM × 1~2년 내 점유 가능 비율(1~5%)
```

### 개인(B2C) — 예시 프레임 (수치는 가정)
- 잠재: 국내 활동 주식 투자자 규모(공개 통계 재확인 필요) 중 "정보 통합·기록" 니즈 세그먼트.
- ARPU 가정: 프리미엄 월 구독 × 전환율.
- → **SOM은 초기 좁은 세그먼트(미·국내 병행 + 기록 성향)에 집중**하여 산정 권장.

### 기업(B2B) — 상대적으로 명확·고ARPU
- 잠재: 국내 상장사 IR·전략기획 조직 + 리서치/운용사. (기업당 좌석 단위 과금)
- ARPU 가정: B2C 대비 수십 배 → **매출 기여는 B2B 우선**이 합리적.

> **결론:** 개인은 볼륨(획득·리텐션 검증), 기업은 매출(고ARPU·명확한 ROI). IR 관점에서 **초기 성장 엔진은 B2B**, 개인은 브랜드·데이터·상향 판매 채널로 포지셔닝. 구체 수치는 실사용 확정 통계로 별첨 예정.

## 9. 기술 아키텍처

```
[사용자 브라우저 · SPA(index.html)]
        │  HTTPS
        ▼
[Supabase]
  ├─ Auth (JWT, 계정 유형 분기)
  ├─ Postgres (user_data, api_keys, security_events, user_profiles)
  │     └─ RLS(행 수준 보안) · JSONB(일지·철학·포트폴리오)
  └─ Edge Functions (Deno)
        ├─ proxy-api      : API 키 암호화 저장/복호화, 외부 API 프록시(CORS 우회)
        └─ market-data    : 시세·뉴스·월가레이팅(fh-call), 공유키 로직
        ▼
[외부 API] DART · Finnhub · Naver · Yahoo Finance · CoinGecko · Google Gemini
```

- **프론트:** 단일 파일 SPA(빌드리스), GitHub Pages 정적 호스팅 → 인프라 비용 최소.
- **백엔드:** Supabase(서버리스) — 초기 고정비 낮고 사용량 기반 확장.
- **보안:** JWT 서명 검증(fail-closed), CORS 화이트리스트, AES-256-GCM 키 암호화, XSS 전수 방어, 보안 이벤트 로깅.
- **비용 구조 강점:** 정적 호스팅 + 서버리스 → **저CAPEX·가변OPEX**로 초기 런웨이 효율 높음. AI만 사용량 비례 원가(요금제로 전가).

## 10. 개발 트랙션 (실행력 증빙)

- **10회 이상 개발 세션**으로 전략 수립 → 인프라(서버 키 이관) → 디자인 시스템 → 안정화(XSS·상태복원) → AI → 계정별 저장 → 교착 근본 해소 → 캘린더 어닝콜 → 경쟁사 BM까지 단계적 완주.
- **실환경 검증 문화:** 주요 변경마다 브라우저 자동화로 라이브(github.io)+로컬 end-to-end 테스트. (본 IR 문서도 배포본 전 기능 실측 후 작성.)
- **난제 해결 이력:** supabase-js 초기화 교착(lock 영구 점유) 근본 해소, KOSDAQ 시세 오류, 월가 레이팅 경로 제한, DART 3대 제약 우회 — **실전 문제를 코드로 끝까지 해결**.

> IR 시사점: 소규모 팀이 **기획–개발–검증–배포 사이클을 빠르게 반복**하고 있음. 자본 투입 시 로드맵 실행 속도의 예측 가능성이 높음.

## 14. 투자 요청 (The Ask)

- **현 단계:** MVP 배포 완료, 수익모델 확정, B2B 지불 근거 명확. **결제 게이트·B2B 검증 직전 단계**.
- **요청:** 시드 투자 — ① B2B 파일럿(상장사 IR팀) 유료 검증 ② 결제·이메일·공용 AI 인프라 완성 ③ 개발/세일즈 리소스.
- **기대 마일스톤:** 유료 전환 게이트 가동 → 첫 B2B 유료 계약 → NSM·전환 지표로 PMF 근거 확보.
