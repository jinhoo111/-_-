# 개발 백로그 — 경쟁사 모니터링 v1.1 (이후 고도화)

> 상태: **대기(승인됨, 미개발)**. v1(공시 레이더 + 홈 브리핑 카드 + Freemium + 통화 토글)은 배포 완료.
> 우선순위 순.

## 1. 키 없이 AI (공용 Gemini 유료 키)
- **목표**: Pro(승인 기업) 사용자가 Google AI 키를 직접 넣지 않아도 AI 브리핑·요약 사용.
- **방식**: `proxy-api` Edge Function에 **관리자 공유 googleai 키** 적용 로직 추가 — 기존 `market-data`의 Finnhub 공유키 패턴(`shared_finnhub_key` / `getAdminKeys` / `admin-save-finnhub-key`) 이식.
  - `user_profiles.shared_googleai_key`(관리자 행) 컬럼 + admin 저장/삭제/상태 액션.
  - 승인 기업 요청 시 서버가 공유 키 적용(개인 키 폴백). 클라 `proxyApiCall('googleai',…)` 경로에서 승인기업은 키 미보유여도 동작.
- **주의**: Gemini **유료(pay-as-you-go)** 티어여야 입력·출력이 학습에 안 쓰임(기업 데이터 보호). EEA/UK/스위스는 무료티어 클라 제공 불가.
- **배포**: 별도 Edge 배포 필요.

## 2. 주간 리포트 내보내기 (PDF / 노션)
- 홈 브리핑 카드 `[PDF]` 버튼 활성화 + 레이더 주간 종합을 리포트 문서로.
- PDF: `window.print()` 전용 스타일(1차) → 추후 서버 렌더. 노션: 사용자 노션 연동(OAuth) 또는 마크다운 복사.

## 3. 이메일 다이제스트 + 실제 결제 게이트
- **이메일**: Supabase Edge + 메일 제공자(예: Resend)로 주간 경쟁사 다이제스트 발송(옵트인). 스케줄러(cron) 필요.
- **결제/plan**: 현재 `_isPro = 승인기업 프록시`를 실제 요금제로 전환. `user_profiles.plan`(free/pro) 컬럼 + 결제 연동(예: 토스페이먼츠/Stripe) + 서버 검증. 게이트 지점: `renderCompetitorBrief`·`briefRadarSignal`·경쟁사 등록 상한.
