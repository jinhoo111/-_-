## 세션 2 — 기반 강화 Phase 1: API 키 서버 이관 (Supabase)

### 배경
- 기존 구조: API 키를 브라우저 `localStorage`에 평문 저장 → 노출 위험
- 목표: 서버 측(Supabase)에 AES-256-GCM 암호화 후 저장, 클라이언트는 키를 직접 보유하지 않음

### 인프라 구축
1. **Supabase 프로젝트 생성** — `investment_news service`
2. **DB 테이블 생성** (`api_keys`)
   ```sql
   create table api_keys (
     service text primary key,
     encrypted_key text not null,
     iv text not null,
     updated_at timestamptz default now()
   );
   ```
3. **Edge Function 배포** — `proxy-api` (`supabase/functions/proxy-api/index.ts`)
   - AES-256-GCM 암호화/복호화 구현 (Deno Web Crypto API)
   - 액션 4종: `store-key` / `key-exists` / `delete-key` / `call`
   - 서비스 2종: `finnhub` (REST 프록시), `googleai` (Gemini API 프록시)
4. **환경변수** `ENCRYPTION_SECRET` Supabase Dashboard에서 설정
5. **인증**: Supabase Publishable Key를 `apikey` + `Authorization: Bearer` 헤더로 전달

### index.html 변경 사항
```js
// 추가된 전역 상수
const SUPA_URL = 'https://yijkwuiqnviapztqskak.supabase.co/functions/v1/proxy-api';
const SUPA_KEY = 'sb_publishable_xa9hIIRRb6_YWc8MGWNhlA_3Ch6NAiF';
const _sh = {'Content-Type':'application/json','apikey':SUPA_KEY,'Authorization':'Bearer '+SUPA_KEY};

// 추가된 함수
storeApiKey(service, key)     // 서버에 암호화 저장
proxyApiCall(service, path, params)  // 서버 경유 API 호출
_deleteApiKeyOnServer(service)       // 서버 키 삭제
resetGoogleAiKey()                   // Google AI 키 초기화 UI
```
- `FKEY` / `CKEY`: 실제 키 대신 `'***'` (flag-based) 저장
- Finnhub 직접 호출 8곳 → `proxyApiCall('finnhub', ...)` 로 변경
- Google AI 직접 호출 → `proxyApiCall('googleai', ...)` 로 변경
- `connectAPI()` → 키 직접 검증 후 `storeApiKey('finnhub', ...)` 호출
- `savePnKey()` → async 변환, `storeApiKey('googleai', ...)` 호출

### 트러블슈팅
| 문제 | 원인 | 해결 |
|------|------|------|
| Edge Function 파일명 오류 | 함수 파일을 `index.ts` 외 이름으로 생성 | 파일명을 `index.ts`로 변경 |
| CORS 파싱 오류 | 긴 문자열 붙여넣기 중 `content-type` 앞에 개행 삽입 | 별도 const로 분리 |
| 401 Unauthorized | Supabase 기본 JWT 검증이 익명 요청 차단 | Publishable Key를 `apikey` + `Authorization` 헤더에 추가 |

## 세션 3 — 기반 강화 Phase 2: 디자인 시스템 토큰화

### 배경
- 하드코딩된 색상·폰트 값이 CSS + JS 인라인 스타일 전체에 산재
- 단일 변경 포인트 없이는 테마 변경, 다크모드, 브랜드 조정 불가

### 작업 내용

#### 1. CSS 커스텀 프로퍼티 `:root` 블록 신설
총 **90개 토큰** 정의:
- 색상 토큰 78개 (Text 10 / Surface 8 / Border 6 / Success 9 / Error 5 / Warning 8 / Info 6 / Accent 11 / Trade 3 / Broker 9 / 나머지 3)
- 타이포그래피 10개 (`--text-2xs` ~ `--text-3xl`)
- 간격 8개 (`--space-1` ~ `--space-8`)
- 보더 레디어스 7개 (`--radius-sm` ~ `--radius-full`)

#### 2. 전량 토큰 치환
- 파일 전체에서 **72개 고유 hex 색상값** → `var(--color-xxx)` 참조
- **10가지 font-size 값** → `var(--text-xxx)` 참조
- CSS 클래스, HTML 속성, JS 템플릿 리터럴 (`TAG`, `ACCT_COLOR`, `renderStocks` 등) 모두 포함
- 치환 후 파일 내 잔여 raw hex: **0개** (`:root` 정의 제외)

#### 3. 접근성 (WCAG 2.1)
- `.btn`, `.fbtn`, `.tbtn`, `.nav-tab`, `.itab`, `.memo-vtab` → `min-height: 44px` (터치 타겟 2.5.5)
- `:focus-visible` 포커스 링 (`2px solid var(--color-info)`)
- 아이콘 전용 버튼 `aria-label` 추가 (↻ 새로고침, ✕ 삭제/취소, ✓ 저장)

#### 4. 다크모드 준비
- `@media (prefers-color-scheme: dark)` 블록 추가
- Surface 8 / Border 4 / Text 6개 토큰 오버라이드 정의
- Semantic 색상(success/error/warning/info)은 유지

#### 5. 산출물
- `designsystem_richbuild_v2.md` — 전체 토큰 레퍼런스 문서
