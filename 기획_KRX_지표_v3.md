# 기획 — KRX 공식 데이터로 만드는 투자 지표 v3

> 선행: `기획_수급페이지_기관매매_v2.md`(Phase 1~4 완료). 이 문서는 남겨둔 **KRX 키 활용**.
> 상태: **기획**. 필드 검증은 `KRX_KEY` 시크릿 등록 후 실측 예정.
> 확정: ① **시장 온도계**(투자 심리) ② **종목 지표 카드**(개별 종목) — 둘 다 **수급 페이지에 통합**.

---

## 1. 전제 — KRX가 주는 것과 못 주는 것 (2026-07-11 실측)

| 구분 | 내용 |
|---|---|
| ✅ 제공 | 지수 일별시세(KOSPI·KOSDAQ·KRX 시리즈), **주식 일별매매정보(전 종목)**, 종목기본정보, ETF/ETN/ELW, 채권, **선물·옵션 일별매매정보**, 일반상품(금·석유·배출권) |
| ❌ 미제공 | **투자자별 수급(기관·외국인)** — 서비스 자체가 없음. 계속 네이버 사용 |
| ❌ 미제공 | 실시간 시세 — 전부 **일 단위 확정 데이터** (장중 갱신 없음) |
| 한도 | 하루 10,000 호출 — 우리 설계(하루 수 콜)엔 차고 넘침 |

**핵심 이점**: `주식 일별매매정보`는 **한 번 호출하면 그날 전 종목이 통째로** 옵니다.
→ 2,700종목 지표를 만드는 데 **하루 1콜**이면 충분합니다. 네이버 종목별 순회(300콜)와 차원이 다릅니다.

## 2. 기능 ① 시장 온도계 — "지금 들어가도 되나?"

수급 페이지 최상단 요약 배너. 개인이 가장 궁금해하는 질문에 **숫자로** 답합니다.

| 지표 | 계산 | 소스 | 읽는 법 |
|---|---|---|---|
| **거래대금** | 코스피+코스닥 총 거래대금, 20일 평균 대비 % | 지수 일별시세 | 평균 대비 급증 = 과열/이벤트 |
| **상승·하락 종목 수** | 전 종목 등락 집계 (상승/하락/보합) | 주식 일별매매정보 | 지수는 올랐는데 하락 종목이 많다 = 소수 종목 착시 |
| **풋콜 비율(P/C)** | 풋 거래량 ÷ 콜 거래량 | 옵션 일별매매정보 | 높을수록 공포(하락 베팅), 낮을수록 탐욕 |
| **외국인·기관 순매수** | 합산 순매수 | 네이버(기존) | 수급 주체 방향 |

**설계 원칙(중요)**: 이 넷을 뭉뚱그린 **자체 "온도 점수"는 만들지 않습니다.** 근거 없는 종합 점수는
투자 판단을 오도합니다. 지표를 **각각 그대로** 보여주고, 각 지표에 *중립적인 해석 라벨*
(예: 거래대금 "20일 평균 +38% — 평소보다 활발")만 붙입니다.

## 3. 기능 ② 종목 지표 카드 — "이 종목은 지금 어떤 상태인가"

내 보유 종목 / 모니터링 종목 / 수급 랭킹에서 종목 클릭 시 표시.

| 지표 | 계산 | 왜 유용한가 |
|---|---|---|
| **시가총액 · 상장주식수** | KRX 원본 | 규모 파악, 유통물량 감 |
| **거래대금 순위** | 당일 전 종목 정렬 | 시장 관심도 (거래대금이 곧 관심) |
| **회전율** | 거래량 ÷ 상장주식수 | 손바뀜 속도. 급등주 과열 판단 |
| **52주 고·저 대비 위치** | (현재가−52주저)÷(52주고−52주저) | 0%면 바닥권, 100%면 신고가권 |
| **지수 대비 상대강도(RS)** | 종목 20일 수익률 − KOSPI 20일 수익률 | 시장보다 강한가 약한가 |
| **외국인·기관 수급** | 기존 네이버 데이터 결합 | 누가 사는가 |

**차별점**: 이 지표들 자체는 증권사 앱에도 있습니다. 우리만의 값어치는 **내 포트폴리오와 결합**입니다.
- "내 보유 종목 중 **52주 고점 대비 −30% 이하**인 종목"
- "내 종목 중 **기관 연속 순매도 + RS 마이너스**" → 이미 만든 수급 경고와 연결

## 4. 데이터 파이프라인

```
[일 1회 배치 · 장 마감 후]
  KRX 주식 일별매매정보(basDd=오늘)  ← 1콜로 전 종목
  KRX 지수 일별시세                   ← 1콜
  KRX 옵션 일별매매정보               ← 1콜
        │
        ├─ 시장 온도계 지표 계산 → api_cache (key: krx:market:YYYYMMDD)
        └─ 종목별 파생지표 계산 → krx_metrics 테이블 (종목당 1행)
              시총·상장주식수·거래대금·회전율·52주고저·20일수익률·RS

[조회] 사용자 → Edge → 캐시/테이블 즉시 반환 (외부 호출 0)
```

**52주 데이터 확보**: 최초 1회만 과거 250영업일을 순회 적재(250콜, 한도 10,000 대비 여유).
이후에는 **하루 1콜**로 최신일만 덧붙이고 52주 창을 굴립니다.

**저장 설계**: 250일 × 2,700종목 원본을 다 넣으면 67만 행이라 무료 티어에 부담.
→ 원본은 저장하지 않고 **종목당 파생지표 1행**(약 2,700행)만 유지. 52주 고·저는 롤링 갱신.

## 5. 시세 소스 교체(부수 효과)

지금 국내 시세는 네이버 비공식 API 의존 → **KRX 공식 일별매매정보(종가 기준)로 대체 가능**.
다만 KRX는 **장중 실시간이 없으므로**, 장중에는 네이버, 마감 후 확정치는 KRX로 쓰는 **이원 구조**가 현실적.
(포트폴리오 평가금액은 확정 종가가 더 정확 → 손익 계산 신뢰도 상승)

---

## 6. 개발 순서

1. **KRX_KEY 시크릿 등록 + 필드 실측** ← *지금 필요*
2. 배치 액션 `krx-refresh` (전 종목·지수·옵션 1콜씩) + `krx_metrics` 테이블
3. **시장 온도계** 배너 (수급 페이지 상단)
4. **종목 지표 카드** + 포트폴리오 결합 경고
5. (선택) 시세 이원화(장중 네이버 / 마감 후 KRX 확정)
