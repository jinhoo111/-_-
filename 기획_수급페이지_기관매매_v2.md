### ⚠️ 반드시 인지할 한계 — 미국 "기관 매매"
13F는 **분기 종료 후 45일 이내** 제출이다. 즉 무료로는 **"지금 외국 기관이 뭘 사는가"를 알 수 없다.**
- 얻을 수 있는 것: **분기 단위 포지션 변화**("버크셔가 지난 분기 애플 축소") — 지연되지만 대가들의 방향성
- 대신 **Form 4(내부자)는 거래 후 2영업일 내 신고 → 거의 실시간**. 개인 투자자에게 체감·신호 가치가 더 큼 → **먼저 개발**

### KRX 발급 키의 활용처 (수급 아님)
투자자별 수급엔 못 쓰지만 다음에 유용하다. 후순위 개선 과제로 남긴다.
- **종목 유니버스 구성**: 종목기본정보 → 코스피/코스닥 전체 목록(랭킹 대상 확정용)
- **국내 시세 공식화**: 현재 네이버 의존 → KRX 일별매매정보로 대체 가능 (하루 10,000회 한도로 충분)

## 2. 신규 '수급' 페이지 구조

```
[💰 수급]  ← 상단 탭 신설
 ├─ 🇰🇷 국내 수급          (Phase 1)
 │   ├─ 기관 순매수 TOP 20 / 순매도 TOP 20
 │   ├─ 외국인 순매수 TOP 20 / 순매도 TOP 20
 │   └─ 내 보유 종목의 수급  ← 포트폴리오 연계(차별화 포인트)
 ├─ 🇺🇸 미국 내부자(Form 4) (Phase 2)
 │   ├─ 최신 내부자 거래 피드 (전체 / 내 보유·관심만)
 │   └─ 매수·매도 방향 필터
 └─ 🏛️ 기관 보유(13F)      (Phase 3)
     └─ 주요 기관 분기별 포지션 변화 (버크셔·브리지워터 등)
```

## 3. Phase 1 — 국내 기관·외국인 순매수 (핵심)

### 3-1. 데이터 흐름
```
[배치] 평일 16:10 KST (장 마감 후)
  GitHub Actions cron → Edge `flow-kr-refresh`
    → 유니버스 N종목(시총 상위 300) 순회: m.stock.naver.com trend
    → 기관/외국인 순매수 정렬 → 상·하위 20 산출
    → api_cache 저장 (key: flow:kr:rank:YYYYMMDD)
[조회] 사용자 → Edge `flow-kr-rank` → api_cache 즉시 반환 (외부 호출 0)
```
- 수급은 **일 단위 확정 데이터** → 하루 1회 갱신으로 충분. 사용자 요청이 아무리 많아도 외부 호출 0.
- 배치 실패/미실행 시: 첫 조회가 온디맨드로 계산(수십 초) 후 캐시 → **화면이 비지 않음**.
- 네이버 호출 300건은 100ms 간격으로 예의 있게(rate-limit 회피).

### 3-2. Edge 액션 (market-data)
| 액션 | 인증 | 설명 |
|---|---|---|
| `flow-kr-rank` | 불필요(공통 데이터) | 캐시된 랭킹 반환. 비회원에게도 공개 → **가입 유입 자산** |
| `flow-kr-stock` | 필요 | 특정 종목 수급 상세(일별 추이). TTL 1시간 |
| `flow-kr-refresh` | 배치 전용(시크릿 헤더) | 랭킹 재계산 |

### 3-3. 소스 추상화 (KIS 교체 대비)
Edge 내부에 `getInvestorFlow(code) → {foreign, organ, individual}` 인터페이스를 두고 네이버 구현을 주입.
KIS 발급 시 구현체만 교체하면 화면·캐시·랭킹 로직은 그대로.

### 3-4. 차별화 포인트
단순 랭킹은 네이버에도 있다. **우리만 할 수 있는 것**은 포트폴리오 연계다.
- "내 보유 종목 중 **기관이 3일 연속 순매도**한 종목: SK하이닉스" → 투자 일지/알림으로 연결
- 홈 브리핑 카드에 "오늘 기관 수급, 내 종목 기준" 한 줄 요약

## 4. Phase 2 — 미국 내부자(Form 4)

v1 문서 §3의 Tier A/B 설계를 그대로 따른다.
- **Tier A(백엔드 0)**: `efts.sec.gov` + `data.sec.gov/submissions` (CORS 개방) → 브라우저 직접 호출.
  회사·내부자명·직위·신고일 피드. 티커→CIK는 `company_tickers.json` 캐시.
- **Tier B(Edge 필요)**: Form 4 XML 파싱 → **매수(P)/매도(S)·수량·단가·금액**. `sec-form4` 액션.
- SEC 요건: **User-Agent 필수**, 10 req/s 제한 → 서버 캐시(TTL 10분) 필수.

---

## 5. Phase 3 — 13F 기관 보유

- 대상 기관을 큐레이션(버크셔·브리지워터·아크 등 10~20곳) → 분기별 보유 XML 파싱 → **직전 분기 대비 증감**.
- "이번 분기 신규 편입 / 전량 매도" 하이라이트. TTL 하루(분기 데이터라 사실상 정적).

---

## 6. Phase 4 — 기업 모니터링 해외 확대

현재: DART(한국 상장사)만. 확대 후: **미국 종목도 동일 카드에서 모니터링**.

| 항목 | 한국(현행) | 미국(신규) |
|---|---|---|
| 종목 식별 | 종목코드 → `corp_map.json`(corp_code) | 티커 → CIK (`company_tickers.json`) |
| 공시 | DART `list.json` | SEC `submissions` (8-K·10-Q·10-K) |
| 시세 | 네이버/KRX | Finnhub `quote` (이미 사용 중) |
| 지분변동 | DART majorstock·elestock | SEC Form 4 · 13D/G |

- 모니터 검색창에서 티커 입력 시 미국 종목으로 인식 → 카드 추가.
- 레이더(시그널 분류)도 8-K 항목코드 기반으로 확장 가능(예: Item 5.02 경영진 교체).

## 7. 개발 순서 제안

1. **Phase 1 국내 수급** — 사용자 요청의 핵심, 실측 완료, 무료·즉시. 신규 수급 페이지 골격 포함.
2. **Phase 2 미국 Form 4 (Tier A)** — 백엔드 0, 빠른 확장.
3. **Phase 4 해외 기업 모니터링** — 기존 모니터 확장.
4. **Phase 3 13F** — 가치는 있으나 지연 데이터, 마지막.
5. (선택) KRX 키로 국내 시세·유니버스 공식화.
